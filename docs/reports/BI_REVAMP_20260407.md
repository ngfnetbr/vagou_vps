## Objetivo
Reestruturar o módulo de BI para:
- Centralizar o modelo analítico em uma camada de dados dedicada (views/funções).
- Otimizar consultas (menos agregação no client) e índices.
- Implementar filtros dinâmicos e consistentes em todos os visuais (gráficos, tabelas e KPIs).
- Manter compatibilidade com ambientes ainda sem a migração aplicada (fallback client-side).

## Mudanças de Banco (Supabase)
Arquivo: [20260407000000_bi_module_revamp.sql](file:///c:/Users/User/Desktop/backup%2000h42%2021-03Vagou/Vagou/vagou/supabase/migrations/20260407000000_bi_module_revamp.sql)

### Camada analítica (arquitetura de dados)
- View `public.bi_criancas_dim`: normaliza colunas usadas no BI (bairro, CEP) e resolve o “turno/período” via `valores_campos_custom`.
- View `public.bi_criancas_zonas`: resolve zonas por inscrição usando:
  - `criancas.zona_atendimento_id` (quando preenchido) como fonte principal.
  - Fallback por regras de `zonas_atendimento.bairros` e `zonas_atendimento.ceps` (quando `zona_atendimento_id` é nulo).
- Função base `public.bi_filtrar_criancas(...)`: aplica todos os filtros de forma consistente e vira fonte única para agregações.

### Agregações (RPCs)
Todas as funções abaixo são `SECURITY DEFINER` e retornam vazio quando o usuário não é admin (`is_admin(auth.uid())`).

- `public.bi_get_novas_inscricoes_mensal(...)`
- `public.bi_get_status_distribuicao(...)`
- `public.bi_get_demanda_por_zona(...)`
- `public.bi_get_demanda_por_bairro(...)`
- `public.bi_get_demanda_por_turno(...)`
- `public.bi_get_demanda_por_cmei(...)`
- `public.bi_get_media_dias_fila_por_cmei(...)`
- `public.bi_get_convocados_por_cmei(...)`

Parâmetros comuns:
- `p_inicio timestamptz`: início do recorte (null = sem recorte).
- `p_statuses text[]`: lista de status (null = todos).
- `p_zona_id uuid` + `p_apenas_sem_zona boolean`: filtro por zona (ou “somente sem zona”).
- `p_bairro text`: busca por bairro (normalizada).
- `p_turno text` + `p_apenas_sem_turno boolean`: filtro por turno (ou “somente não informado”).
- `p_cmei_id uuid`: filtro por CMEI (considera preferência 1/2/3 e CMEI atual).

### Performance
- Novos índices focados no BI:
  - `criancas(status, created_at desc)`
  - `criancas(created_at desc)`
  - `criancas(cmei1_preferencia)`, `criancas(cmei2_preferencia)`, `criancas(cmei3_preferencia)`
  - `criancas(cmei_atual_id)`
  - `criancas(zona_atendimento_id)`
  - índice funcional de `bairro` normalizado para buscas.
- Otimização das RPCs de ocupação:
  - `public.get_ocupacao_cmeis()` e `public.get_ocupacao_turmas()` foram reescritas para usar `LEFT JOIN + GROUP BY` (evita subqueries correlacionadas por linha).
